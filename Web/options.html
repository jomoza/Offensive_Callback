What is 5ELG Multi-Protocol Intelligence Collection Server

5ELG (5ELG — brow5er ExfiLtration and finGerprinting) is a passive multi-protocol intelligence collection server. It captures, fingerprints, and cross-correlates inbound traffic across HTTP(S), WebSocket, DNS and ICMP simultaneously.

🌐 HTTP(S) · Dealer Engine

Serves Dealers — configurable JS/PHP/PS payloads deployed on target pages. Each dealer silently collects and transmits telemetry to this server in real time.

What it captures
  • Fu — SHA-256 browser fingerprint (canvas, fonts, WebGL)
  • Fb — Canvas hash for cross-session device profiling
  • Jd — Device JSON: screen, timezone, plugins, languages
  • Screenshots, POST form captures, localStorage dumps
  • Network topology via internal LAN scanner

🔌 WebSocket · Real-time C2

Maintains persistent bidirectional channels with active dealer sessions. Shares the same port as HTTP — no extra configuration needed if WS_SERVER=true.

Capabilities
  • Real-time telemetry streaming from browser to server
  • Live JS command injection into active sessions
  • Browser-side LAN network scanner results
  • Keepalive beaconing for long-lived sessions
  • Cross-tab session correlation via shared Fu

📡 DNS · Covert Channel

Operates as authoritative DNS for your registered domain. Every lookup for your domain passes through 5ELG — enabling host discovery, exfiltration detection and covert C2 via encoded subdomains.

What it enables
  • Detection of DNS-based data exfiltration attempts
  • Covert C2 via encoded subdomain queries
  • Host discovery — who resolves your domain
  • Relay mode: unknown queries forwarded to 1.1.1.1
  • Custom A / CNAME / MX / TXT record management

DNS_PORT=53 (root required)
DNS_RELAY=true → DNS_RELAY_IP
Logged as
DNS.LOG
Manage DNS records in the DNS Server tab. Add dealer domains to enable sub-domain routing per campaign.
🏓 ICMP · Ping Listener

Passive raw-socket listener for inbound ICMP echo packets. Requires root. Captures pings from recon scanners, port scanners and covert ICMP beacons — no active port needed.

What it captures
  • Host discovery pings from reconnaissance scanners
  • Covert ICMP-based beacon callbacks
  • Source IP, timestamp and payload size
  • Cross-correlation with HTTP/DNS hits from same IP
ENV vars
ICMP_LISTENER=true
Requires root privileges
Logged as
ICMP.LOG
ICMP data is cross-referenced with HTTP and DNS entries to identify IPs that hit multiple protocols — a strong indicator of targeted scanning or reconnaissance.

Dealers
What are Dealers?

Dealers are payloads deployed on target surfaces that silently collect telemetry and send it back to 5ELG. Each incoming request is fingerprinted — IP resolved, browser hashed (Fu/Fb), device profiled (Jd) — and logged to the database. The dealer type controls what gets served and how the response behaves.

XSS — JS payload

XSS serves a JavaScript fingerprint agent (SHA-256 Fu/Fb, device data, WebRTC IP, POST intercept). Uses the built-in dl.js template unless a custom .js file is configured.
XSS-XL is the full agent — same as XSS but also takes a canvas screenshot and streams the page HTML back. Uses dl.full.js.

HTML — page dealer

Serves a full HTML page with the tracking script injected before </body>. Use a custom file for branded phish pages, fake login portals or lure documents. Falls back to the default dealer.html template if no file is set.

URL — landing / redirect

URL serves a tracker landing page (with optional file) on GET. With loot mode enabled it also redirects to a target after capture. POST returns 200 silently.
REDIR always redirects — useful as an invisible hop in link chains.

Inactive dealers are silently ignored — all requests return a 1×1 transparent pixel with no DB write. Active dealers bump total_requests and last_time_request on every hit. The dealer URI is auto-constructed from urls[] + a random alias at serve time. Callback endpoint: — port 80/443.
