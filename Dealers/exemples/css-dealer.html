Mechanism

How CSS Telemetry Works

Browsers process CSS before executing JavaScript — and when they do, they fire network requests for fonts, backgrounds, and images. Every one of those requests hits a server that logs the parameters.

01
@font-face local() fallback. The browser checks if a font exists locally. If not found, it fetches the url() fallback — a server endpoint that records the probe signal. Works even with JavaScript completely disabled.
02
@supports() feature detection. CSS properties like -webkit-app-region only exist in Blink/Chrome. -moz-appearance only in Gecko/Firefox. When the condition is true, a background-image URL is resolved — identifying the rendering engine.
03
@media queries. Screen width, DPR, color gamut, prefers-color-scheme, prefers-reduced-motion, pointer type — each condition fires a different probe URL when matched.
04
Extension fingerprinting. Ad blockers hide elements with classes like .ads or .advertisement. By placing a probe element with those classes and a sibling probe, we detect whether the first was suppressed.
Example

The Probe Pattern

/* OS Detection — font probe */
@font-face {
    font-family: '__probe_win_segoe';
    src: local('Segoe UI'),              /* exists on Windows → no request */
         url('/log?u=font(Segoe-UI)&b=OS-Windows')  /* NOT on Windows → request fires */
         format('truetype');
}

/* Browser Engine Detection */
@supports (-webkit-app-region: inherit) {
    .probe {
        background-image: url('/log?u=engine(Blink)&b=Browser');
    }
}

/* Hardware / Display */
@media (color-gamut: p3) {
    .probe {
        background-image: url('/log?u=display(P3-WideGamut)&b=Display');
    }
}

Research Context

Why CSS Fingerprinting Matters

→
Bypasses JavaScript blockers. NoScript, JavaScript disabled, Tor Browser's JS toggle — none of these stop CSS from loading. Probes fire at the stylesheet evaluation phase, before any JS engine is even invoked.
→
Invisible to most privacy tools. uBlock Origin, Privacy Badger, and similar extensions block known tracker domains — but a first-party endpoint on the same domain as the page cannot be blocked without breaking the page itself.
→
Survives incognito / private mode. Incognito does not change the OS, installed fonts, screen hardware, or browser engine. All font and @supports probes fire identically.
→
Documented by EFF, Mozilla & academics. CSS-based fingerprinting is covered in Panopticlick research, the Am I Unique? project, and is a known vector in OWASP's browser fingerprinting taxonomy.
